# UI Contracts: FrontGuidKit 메인페이지

**Feature**: 003-main-page
**Date**: 2026-03-09

## C-001: Hero Section

**위치**: 메인페이지 Body 최상단
**구조**:
```
┌──────────────────────────────────────────┐
│         </> FrontGuidKit                 │  ← 제목 (큰 볼드체)
│  프론트엔드 개발 가이드 & AI 프롬프트       │  ← 부제목 (중간 크기)
│  간단한 설명 텍스트                        │  ← 설명 (muted 색상)
└──────────────────────────────────────────┘
```

**렌더링 규칙**:
- 제목: `</> FrontGuidKit` — 코드 스타일 `</>` + 프로젝트명
- 부제목: `프론트엔드 개발 가이드 & AI 프롬프트`
- 설명: 프로젝트의 목적을 1~2문장으로 설명
- 중앙 정렬, 상하 여백 충분히 확보

## C-002: Category Card Grid

**위치**: Hero 섹션 아래, 카테고리별 반복
**구조**:
```
┌── 카테고리 섹션 ────────────────────────────┐
│ [아이콘] 카테고리명                          │  ← 섹션 타이틀
│ ┌──────┐ ┌──────┐ ┌──────┐                 │
│ │ Card │ │ Card │ │ Card │                 │  ← 3열 그리드
│ │ ┌──┐ │ │ ┌──┐ │ │ ┌──┐ │                 │
│ │ │PH│ │ │ │PH│ │ │ │PH│ │                 │  ← Placeholder 영역
│ │ └──┘ │ │ └──┘ │ │ └──┘ │                 │
│ │ Name │ │ Name │ │ Name │                 │  ← 컴포넌트명
│ └──────┘ └──────┘ └──────┘                 │
└─────────────────────────────────────────────┘
```

**렌더링 규칙**:
- 섹션 타이틀: 카테고리 아이콘(Lucide) + 카테고리명
- 카드 그리드: 데스크탑 3열, 태블릿 2열, 모바일 1열
- 카드 내용: 상단에 와이어프레임 placeholder 영역, 하단에 컴포넌트명
- 카드 클릭 → `/components/[slug]` 이동
- 컴포넌트가 0개인 카테고리: 섹션 자체를 렌더링하지 않음

## C-003: Theme Toggle

**위치**: 헤더(TopBar) 우측
**구조**:
```
┌─ Header ──────────────────────────────────────────┐
│ [Code2] FrontGuidKit  프론트엔드 ...    [Sun/Moon] │
└───────────────────────────────────────────────────┘
```

**렌더링 규칙**:
- 다크모드 시: Sun 아이콘 표시 (클릭하면 라이트로 전환)
- 라이트모드 시: Moon 아이콘 표시 (클릭하면 다크로 전환)
- 아이콘 버튼은 접근성을 위해 `aria-label` 포함
- 전환 시 깜빡임(FOUC) 방지

## C-004: Sidebar Home Item

**위치**: 사이드바 최상단 (카테고리 목록 위)
**구조**:
```
┌─ Sidebar ──────────┐
│ [Home] 홈           │  ← Home 아이콘 + "홈" 텍스트
│ ─────────────────── │  ← 구분선
│ 컴포넌트             │  ← 기존 카테고리 그룹 레이블
│  ▶ 레이아웃          │
│  ▶ 피드백/알림       │
│  ...                │
└─────────────────────┘
```

**렌더링 규칙**:
- Home 아이콘 클릭 → `/` 이동
- 메인페이지(`/`)에서: Home 항목이 active 상태, 모든 Collapsible 접힘
- 상세 페이지에서: Home 항목이 비활성, 해당 카테고리 Collapsible 열림

## C-005: Footer

**위치**: 모든 페이지 하단 (SidebarInset 내부)
**구조**:
```
┌─ Footer (3열) ─────────────────────────────────────┐
│ 링크 모음          │ 저작권 표기        │ 소셜 아이콘  │
│ · GitHub          │ © 2026            │ [GH] [X]    │
│ · 문서             │ FrontGuidKit      │             │
│ · 블로그           │ All rights        │             │
│                   │ reserved.         │             │
└────────────────────────────────────────────────────┘
```

**렌더링 규칙**:
- 3열 그리드: 데스크탑에서 동일 폭 3열, 모바일에서 1열 수직 배치
- 링크 모음: 임시 URL(`#`)으로 3~4개 링크
- 저작권: `© 2026 FrontGuidKit. All rights reserved.`
- 소셜 아이콘: GitHub, X(Twitter) 등 Lucide 아이콘 사용
- 배경색: 본문보다 약간 어두운/밝은 톤 (테마 대응)

## C-006: Header Design Changes

**변경 사항**:
- 헤더 높이: 기존 `h-14` (56px) → `h-[67px]` (약 20% 증가)
- 프로젝트명 폰트: 기존 `text-lg` → `text-xl` (비례 증가)
- 부제목 폰트: 기존 `text-sm` → `text-base` (비례 증가)
