# Feature Specification: FrontGuidKit MVP

**Feature Branch**: `001-frontguidkit-mvp`
**Created**: 2026-03-08
**Status**: Draft
**Input**: PRD-FrontGuidKit.md - 프론트엔드 개발 가이드 & AI 프롬프트 플랫폼 MVP

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 사이드바에서 컴포넌트 탐색 (Priority: P1)

AI 코딩 툴 사용자가 사이드바에서 6개 카테고리(레이아웃, 피드백/알림, 탐색/이동, 콘텐츠 표시, 입력/인터랙션, 데이터 시각화)를 펼쳐 33개 컴포넌트 목록을 탐색하고, 원하는 컴포넌트를 클릭하여 상세 가이드 페이지로 이동한다.

**Why this priority**: 사이드바 네비게이션은 모든 콘텐츠에 접근하는 유일한 경로이므로, 이것 없이는 서비스 자체가 동작하지 않는다.

**Independent Test**: 사이드바만 구현해도 컴포넌트 목록을 확인하고 선택할 수 있어 최소한의 탐색 가치를 제공한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 메인 페이지에 접속한 상태에서, **When** 사이드바의 카테고리(예: "레이아웃")를 클릭하면, **Then** 해당 카테고리 하위의 컴포넌트 목록(Header, Hero, Navbar, Body, Footer)이 펼쳐진다.
2. **Given** 카테고리가 펼쳐진 상태에서, **When** 컴포넌트(예: "Header")를 클릭하면, **Then** 메인 콘텐츠 영역이 해당 컴포넌트의 상세 가이드로 전환되고, 사이드바에서 선택된 항목이 시각적으로 활성 상태로 표시된다.
3. **Given** 이미 펼쳐진 카테고리가 있는 상태에서, **When** 동일한 카테고리를 다시 클릭하면, **Then** 해당 카테고리 목록이 접힌다.
4. **Given** 사용자가 어떤 컴포넌트 가이드를 보고 있는 상태에서, **When** 다른 카테고리의 컴포넌트를 클릭하면, **Then** 페이지 새로고침 없이 메인 영역이 즉시 전환된다.

---

### User Story 2 - 와이어프레임으로 컴포넌트 위치 파악 (Priority: P1)

프론트엔드 입문자가 컴포넌트 상세 페이지에서 와이어프레임 미리보기를 통해, 해당 컴포넌트가 전체 페이지 레이아웃 내 어디에 위치하는지 시각적으로 확인한다.

**Why this priority**: 와이어프레임은 서비스의 핵심 차별점이며, 컴포넌트의 시각적 이해를 가능하게 하는 핵심 기능이다.

**Independent Test**: 와이어프레임만으로도 컴포넌트가 페이지 어디에 위치하는지 이해할 수 있어 독립적인 학습 가치를 제공한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 컴포넌트 상세 페이지에 접속한 상태에서, **When** 와이어프레임 패널을 확인하면, **Then** 전체 페이지 레이아웃이 표시되고 해당 컴포넌트의 위치가 색상 하이라이트로 강조된다.
2. **Given** Header 컴포넌트 페이지에서, **When** 와이어프레임을 확인하면, **Then** 전체 페이지 구조 내에서 상단 영역이 하이라이트되고, 내부 구성 요소(로고, 네비게이션, CTA 버튼 등)가 텍스트 레이블로 표시된다.
3. **Given** 콘텐츠가 아직 준비되지 않은 컴포넌트 페이지에서, **When** 사용자가 해당 페이지에 접근하면, **Then** "콘텐츠 준비 중" 안내 메시지가 표시된다.

---

### User Story 3 - 핵심 용어 학습 (Priority: P2)

비개발 직군(기획자, 디자이너, PM)이 컴포넌트 상세 페이지에서 4개의 핵심 용어 카드를 통해 해당 컴포넌트 관련 프론트엔드 용어를 빠르게 학습한다.

**Why this priority**: 용어 학습은 프로젝트의 교육적 가치를 담당하며, AI 프롬프트 작성의 기반이 된다. 하지만 와이어프레임과 네비게이션이 먼저 있어야 의미가 있다.

**Independent Test**: 용어 카드만으로도 컴포넌트 관련 핵심 용어 4개를 학습할 수 있어 독립적인 교육 가치를 제공한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 컴포넌트 상세 페이지에 접속한 상태에서, **When** 핵심 용어 섹션을 확인하면, **Then** 4개의 카드가 표시되며 각 카드에 영어 명칭과 한국어 설명이 포함된다.
2. **Given** Header 컴포넌트 페이지에서, **When** 핵심 용어 카드를 확인하면, **Then** 예를 들어 "Sticky Header — 스크롤 시 상단에 고정되는 헤더"와 같은 형태로 용어가 표시된다.

---

### User Story 4 - AI 프롬프트 복사 (Priority: P1)

AI 코딩 툴 사용자가 컴포넌트 상세 페이지에서 AI 프롬프트 템플릿을 확인하고 [복사] 버튼을 클릭하여 클립보드에 복사한 후, Claude Code나 Codex 등에 바로 붙여넣기하여 UI 코드를 생성한다.

**Why this priority**: 프롬프트 복사는 서비스의 핵심 가치 제안(Value Proposition)이자 KR2의 측정 대상이다. 사용자가 실질적으로 가져가는 액션이므로 P1이다.

**Independent Test**: 프롬프트 복사 기능만으로도 사용자가 AI 도구에 즉시 활용할 수 있는 프롬프트를 얻어 독립적 가치를 제공한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 컴포넌트 상세 페이지에 접속한 상태에서, **When** AI 프롬프트 패널을 확인하면, **Then** 해당 컴포넌트의 구조, 동작, 스타일이 포함된 프롬프트 템플릿이 표시된다.
2. **Given** 프롬프트 패널이 표시된 상태에서, **When** [복사] 버튼을 클릭하면, **Then** 전체 프롬프트가 클립보드에 복사되고, 버튼 텍스트가 "복사됨"으로 변경된다.
3. **Given** 복사 완료 후, **When** 2초가 경과하면, **Then** 버튼이 원래의 [복사] 상태로 돌아온다.

---

### User Story 5 - 다크 테마 UI 경험 (Priority: P2)

사용자가 다크 테마 기반의 UI로 서비스를 이용하며, 장시간 사용 시에도 눈의 피로를 줄일 수 있다.

**Why this priority**: 다크 테마는 개발자 타겟 사용자의 기대 사항이며 전반적인 UX 품질에 영향을 주지만, 핵심 기능보다는 후순위이다.

**Independent Test**: 다크 테마 적용 여부만으로도 시각적 완성도와 사용자 만족도를 독립적으로 테스트할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 서비스에 접속하면, **When** 페이지가 로드되면, **Then** 다크 테마가 기본으로 적용된 UI가 표시된다.
2. **Given** 다크 테마 상태에서, **When** 모든 페이지를 탐색하면, **Then** 텍스트, 카드, 와이어프레임 등 모든 요소의 가독성이 유지된다.

---

### Edge Cases

- 사용자가 콘텐츠가 아직 준비되지 않은 컴포넌트(MVP 이후 Phase 2 대상)를 클릭했을 때 → "콘텐츠 준비 중" 플레이스홀더를 표시한다.
- 클립보드 API를 지원하지 않는 브라우저에서 복사 버튼을 클릭했을 때 → 폴백으로 프롬프트 텍스트를 선택 상태로 만들어 수동 복사를 유도한다.
- 사이드바에서 동일한 컴포넌트를 연속으로 클릭했을 때 → 불필요한 리렌더링이 발생하지 않아야 한다.
- 브라우저 뒤로가기/앞으로가기 버튼으로 컴포넌트 간 이동이 가능해야 한다.
- 특정 컴포넌트 페이지의 URL을 직접 입력하거나 북마크해서 접근할 수 있어야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 6개 카테고리(레이아웃, 피드백/알림, 탐색/이동, 콘텐츠 표시, 입력/인터랙션, 데이터 시각화)로 구성된 사이드바 네비게이션을 제공해야 한다.
- **FR-002**: 사이드바의 각 카테고리는 클릭하여 펼치거나 접을 수 있어야 하며, 하위 컴포넌트 목록(총 33개)을 표시해야 한다.
- **FR-003**: 선택된 컴포넌트는 사이드바에서 시각적으로 활성 상태를 표시해야 한다.
- **FR-004**: 컴포넌트 클릭 시 페이지 새로고침 없이 메인 콘텐츠 영역이 즉시 전환되어야 한다 (Client-Side Rendering).
- **FR-005**: 각 컴포넌트 상세 페이지에 컴포넌트 제목과 한 줄 설명을 표시해야 한다.
- **FR-006**: 각 컴포넌트 상세 페이지에 전체 페이지 레이아웃 내에서 해당 컴포넌트의 위치를 색상 하이라이트로 표시하는 와이어프레임 미리보기를 제공해야 한다.
- **FR-007**: 와이어프레임 내에 컴포넌트 내부의 세부 구성 요소(로고, 네비게이션, CTA 등)를 텍스트 레이블로 표시해야 한다.
- **FR-008**: 각 컴포넌트 상세 페이지에 4개의 핵심 용어를 카드 형태로 제공해야 하며, 각 카드는 영어 명칭과 한국어 설명을 포함해야 한다.
- **FR-009**: 각 컴포넌트 상세 페이지에 AI 에이전트 구현 요청용 프롬프트 템플릿을 제공해야 하며, 프롬프트에는 구조, 동작, 스타일 등 핵심 항목이 포함되어야 한다.
- **FR-010**: [복사] 버튼 클릭 시 AI 프롬프트 전체가 클립보드에 복사되어야 한다.
- **FR-011**: 복사 완료 시 버튼 상태가 "복사됨"으로 변경되었다가 일정 시간 후 원래 상태로 복귀해야 한다.
- **FR-012**: 다크 테마가 기본 적용된 UI를 제공해야 한다.
- **FR-013**: 데스크톱 환경을 우선 지원하는 반응형 레이아웃을 제공해야 한다.
- **FR-014**: 로그인 없이 모든 기능에 즉시 접근할 수 있어야 한다.
- **FR-015**: 각 컴포넌트 페이지는 고유한 URL을 가지며, 직접 URL 접근 및 브라우저 뒤로가기/앞으로가기가 동작해야 한다.
- **FR-016**: MVP 단계에서 레이아웃 카테고리 5개 컴포넌트(Header, Hero, Navbar, Body, Footer)의 상세 가이드 콘텐츠를 완성해야 한다.
- **FR-017**: 콘텐츠가 준비되지 않은 컴포넌트 페이지에는 "콘텐츠 준비 중" 안내를 표시해야 한다.
- **FR-018**: 상단바에 서비스명("FrontGuidKit")과 서비스 설명("프론트엔드 개발 가이드 & AI 프롬프트")을 표시해야 한다.

### Key Entities

- **카테고리(Category)**: 컴포넌트를 그룹핑하는 상위 분류. 이름, 아이콘, 순서, 하위 컴포넌트 목록을 포함한다. (6개: 레이아웃, 피드백/알림, 탐색/이동, 콘텐츠 표시, 입력/인터랙션, 데이터 시각화)
- **컴포넌트(Component)**: 개별 UI 컴포넌트 가이드의 단위. 이름, 한 줄 설명, 소속 카테고리, 와이어프레임 데이터, 핵심 용어 4개, AI 프롬프트 템플릿을 포함한다. (총 33개)
- **핵심 용어(Term)**: 컴포넌트에 관련된 프론트엔드 용어. 영어 명칭, 한국어 설명을 포함한다. 각 컴포넌트당 4개.
- **AI 프롬프트(Prompt)**: 컴포넌트 구현을 위한 AI 에이전트 요청 템플릿. 구조, 동작, 스타일 항목을 포함하는 텍스트 콘텐츠.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 레이아웃 카테고리 5개 컴포넌트(Header, Hero, Navbar, Body, Footer)의 가이드가 100% 완성되어 와이어프레임, 용어 카드 4개, AI 프롬프트를 모두 포함한다.
- **SC-002**: 사이드바에서 33개 컴포넌트 전체가 탐색 가능하며, 카테고리 펼침/접힘과 컴포넌트 선택이 1초 이내에 반응한다.
- **SC-003**: 프롬프트 복사 버튼 클릭 시 100% 성공적으로 클립보드에 복사되며, 복사된 프롬프트를 AI 코딩 툴에 붙여넣기하면 유효한 UI 구현 요청으로 동작한다.
- **SC-004**: 페이지 방문자의 40% 이상이 프롬프트 복사 버튼을 클릭한다 (KR2 기준).
- **SC-005**: 사용자가 서비스에 접속한 후 원하는 컴포넌트의 프롬프트를 복사하기까지 3단계 이내(카테고리 선택 → 컴포넌트 선택 → 복사 클릭)로 완료할 수 있다.
- **SC-006**: 다크 테마 환경에서 모든 텍스트 요소의 명도 대비가 WCAG 2.1 AA 기준(4.5:1)을 충족한다.
- **SC-007**: 데스크톱 브라우저(Chrome, Firefox, Safari, Edge 최신 버전)에서 레이아웃이 깨지지 않고 정상 표시된다.

## Assumptions

- 한국어 사용자를 1차 타겟으로 하며, 모든 콘텐츠는 한국어로 제공한다.
- 로그인/회원가입 기능은 불필요하며, 모든 콘텐츠는 공개적으로 접근 가능하다.
- 콘텐츠(와이어프레임 데이터, 용어, 프롬프트)는 정적 데이터로 관리하며, 별도의 백엔드 서버나 데이터베이스를 필요로 하지 않는다.
- MVP에서는 라이트 테마 전환 기능 없이 다크 테마만 제공한다.
- 와이어프레임은 실제 UI가 아닌 다이어그램 형태의 시각적 구조도이다.
- AI 프롬프트는 특정 AI 도구에 종속되지 않는 범용적인 형태로 작성한다.
- MVP에서 검색 기능, 프롬프트 커스터마이징, 다국어 지원은 범위에 포함하지 않는다.
- 태블릿/모바일 대응은 MVP 이후 Phase에서 진행한다.
