# FrontGuidKit 메인페이지 기획서

## 1. 서비스 개요
- **한 줄 정의**: FrontGuidKit의 메인 허브 페이지로, 6개 카테고리 33개 컴포넌트를 카드 형태로 한눈에 탐색할 수 있는 랜딩 페이지
- **핵심 가치 제안**: 컴포넌트 가이드의 진입점을 제공하여, 카테고리별 와이어프레임 미리보기를 통해 원하는 컴포넌트에 빠르게 접근
- **핵심 키워드**: `메인페이지`, `카테고리 탐색`, `카드 그리드`, `다크모드`, `반응형 Footer`

## 2. 배경 및 문제 정의

### 2.1 현재 상황
- 현재 메인페이지(`/`)는 `/components/header`로 리다이렉트되어 별도의 랜딩 페이지가 존재하지 않음
- 사용자가 처음 접속했을 때 프로젝트의 전체 구조와 제공 범위를 파악할 수 없음
- 다크모드/라이트모드 전환 기능이 없어 사용자 환경에 맞는 시각적 경험을 제공하지 못함
- Footer가 없어 프로젝트 정보, 링크, 소셜 채널 등의 부가 정보 제공이 불가

### 2.2 문제 정의
- **진입점 부재**: 프로젝트의 전체 카테고리와 컴포넌트를 한눈에 파악할 수 있는 허브 페이지가 없음
- **탐색 제한**: 사이드바를 통해서만 컴포넌트에 접근 가능하여 시각적 탐색 경험이 부족
- **테마 미지원**: 시스템 테마에 따른 자동 모드 전환 및 수동 토글 기능 부재

### 2.3 기존 솔루션의 한계
- 현재 `redirect("/components/header")` 방식은 메인 콘텐츠 없이 단순 우회 처리
- 사이드바 네비게이션만으로는 33개 컴포넌트의 전체 구성을 직관적으로 보여주기 어려움

## 3. 서비스 목표

### 3.1 핵심 목표
- 메인페이지를 통해 6개 카테고리, 33개 컴포넌트를 카드 그리드로 시각화
- 다크모드/라이트모드 테마 시스템 구현 (시스템 설정 자동 감지 + 수동 토글)
- 3열 구조 Footer를 통한 프로젝트 정보 제공

### 3.2 성공 지표 (KPI)
| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 메인페이지 정상 렌더링 | 6개 카테고리 섹션, 33개 카드 표시 | 빌드 및 시각 확인 |
| 테마 전환 동작 | 시스템 감지 + 수동 토글 정상 작동 | 기능 테스트 |
| 반응형 레이아웃 | 모바일/태블릿/데스크탑 정상 표시 | 브라우저 리사이즈 테스트 |
| Footer 렌더링 | 3열 구조 정상 표시 | 시각 확인 |

## 7. 프로젝트 구조

### 7.1 시스템 아키텍처

```
┌─────────────────────────────────────────────────────┐
│                    app/layout.tsx                     │
│  ┌──────────────────────────────────────────────┐   │
│  │ TopBar (헤더) ─── [로고] [부제목] [테마토글]    │   │
│  ├──────────────────────────────────────────────┤   │
│  │ SidebarProvider                               │   │
│  │ ┌──────────┬─────────────────────────────┐   │   │
│  │ │ Sidebar  │  SidebarInset               │   │   │
│  │ │ [Home]   │  ┌───────────────────────┐  │   │   │
│  │ │ [카테고리]│  │ page.tsx (메인/상세)   │  │   │   │
│  │ │  └항목들  │  │                       │  │   │   │
│  │ │          │  │ Hero 섹션              │  │   │   │
│  │ │          │  │ 카테고리 카드 그리드    │  │   │   │
│  │ │          │  │ ...                    │  │   │   │
│  │ │          │  └───────────────────────┘  │   │   │
│  │ └──────────┴─────────────────────────────┘   │   │
│  │ Footer ─── [링크] [저작권] [소셜]              │   │
│  └──────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### 7.2 기술 스택
| 분류 | 기술 | 선정 이유 |
|------|------|-----------|
| Framework | Next.js 16.1.6 (App Router) | 기존 프로젝트 스택, SSG 지원 |
| UI Library | React 19 | 기존 프로젝트 스택 |
| Styling | Tailwind CSS v4 | 기존 프로젝트 스택, 유틸리티 기반 스타일링 |
| Components | shadcn/ui (radix-nova) | 기존 프로젝트 스택, Card 등 UI 컴포넌트 재사용 |
| Icons | Lucide React | 기존 프로젝트 스택, Home/Sun/Moon 등 아이콘 |
| Theme | next-themes | Next.js 공식 권장 테마 라이브러리, 시스템 감지 및 토글 지원 |

### 7.3 데이터 모델
- 기존 정적 TypeScript 객체 기반 데이터 모델 유지 (DB 없음)
- 메인페이지는 `data/categories.ts`와 `data/components.ts`를 그대로 활용

| 엔티티 | 주요 속성 | 메인페이지 활용 |
|--------|-----------|----------------|
| Category | id, name, icon, order, components | 섹션 타이틀 + 아이콘 |
| ComponentData | slug, name, categoryId, wireframe, isReady | 카드 이름, 링크, 미리보기 이미지 |

### 7.4 프로젝트 개발 단계

| 단계 | 목표 | 주요 작업 | 산출물 | 완료 기준 |
|------|------|-----------|--------|-----------|
| Step 1 | 테마 시스템 구축 | `next-themes` 설치, ThemeProvider 적용, 시스템 테마 자동 감지, `html` 태그 동적 클래스 적용 | `components/theme-provider.tsx`, layout.tsx 수정 | 시스템 테마에 따라 다크/라이트 자동 적용 |
| Step 2 | 헤더 개선 | TopBar에 테마 토글 버튼(Sun/Moon 아이콘) 추가, 헤더 폭 20% 확대, 폰트 크기 20% 확대 | `top-bar.tsx` 수정 | 테마 토글 정상 동작, 확대된 헤더 표시 |
| Step 3 | Sidebar Home 아이콘 | Sidebar 상단에 Home 아이콘 + "홈" 메뉴 추가, 클릭 시 `/`로 이동, 메인페이지에서 Collapsible 모두 접힘 | `sidebar.tsx` 수정 | Home 클릭 → 메인페이지 이동, Collapsible 접힘 |
| Step 4 | 메인페이지 구현 | `app/page.tsx`를 메인페이지로 교체, Hero 섹션(제목/부제목/설명), 카테고리별 3열 카드 그리드 | `app/page.tsx`, `components/home/` | 6개 섹션, 33개 카드 정상 렌더링 |
| Step 5 | Footer 구현 | 3열 Footer 컴포넌트 생성(링크/저작권/소셜), layout.tsx에 통합 | `components/layout/footer.tsx`, layout.tsx 수정 | Footer 정상 표시 |
| Step 6 | 전체 폰트 크기 조정 | globals.css에서 base font-size 20% 확대 적용 | `globals.css` 수정 | 전체 텍스트 크기 20% 증가 확인 |
| Step 7 | 통합 테스트 및 빌드 | `npm run build` + `npm run lint` 통과 확인, 반응형 레이아웃 확인 | 빌드 성공 로그 | 빌드/린트 에러 없음 |

## 9. 서비스 제공 내용

### 9.1 서비스 범위 (MVP)
**포함:**
- 메인페이지 Hero 섹션 (제목, 부제목, 설명)
- 6개 카테고리 × 컴포넌트 카드 그리드 (3열)
- 카드 클릭 시 상세 페이지(`/components/[slug]`) 이동
- Sidebar Home 아이콘 및 메인페이지 이동
- 다크모드/라이트모드 시스템 감지 + 수동 토글
- 3열 Footer (링크 모음, 저작권, 소셜 아이콘)
- 헤더/폰트 크기 20% 확대

**포함하지 않는 것:**
- 카드 와이어프레임 실제 이미지 (placeholder로 대체, 추후 첨부)
- 검색 기능
- 사용자 인증/로그인
- 다국어 지원

### 9.2 서비스 운영 계획
- 정적 사이트 생성(SSG) 기반으로 별도 서버 운영 불필요
- Vercel 또는 GitHub Pages를 통한 배포 가능
- `npm run build`로 빌드 검증 후 배포
