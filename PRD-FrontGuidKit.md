# PRD — FrontGuidKit

> 프론트엔드 개발 가이드 & AI 프롬프트 플랫폼

---

## 1. Summary

FrontGuidKit은 프론트엔드 UI 컴포넌트의 구조, 용어, AI 프롬프트를 한 곳에서 제공하는 웹 가이드 플랫폼이다. 프론트엔드 입문자와 AI 코딩 툴 사용자가 컴포넌트 명칭과 구조를 빠르게 파악하고, 바로 복사해서 쓸 수 있는 프롬프트로 원하는 UI를 만들 수 있도록 돕는다.

## 3. Background

### 맥락

Claude Code, Codex, GeminiCLI 같은 AI 코딩 에이전트가 빠르게 보급되면서, 프롬프트만으로 UI를 생성하는 워크플로우가 일상이 되고 있다. 그러나 "Header에 Sticky 속성을 주고 CTA 버튼을 오른쪽에 배치해줘"처럼 정확한 용어를 써야 원하는 결과를 얻을 수 있다. 컴포넌트 명칭과 구조를 모르면 AI에게 제대로 요청할 수 없다.

### 왜 지금인가?

- AI 코딩 툴 사용자가 급증하고 있으나, 프론트엔드 용어를 체계적으로 학습할 수 있는 한국어 리소스가 부족하다.
- 기존 컴포넌트 라이브러리 문서(MUI, Chakra 등)는 개발자 대상이며, 입문자나 비개발 직군이 접근하기 어렵다.
- "시각적 가이드 + 용어 + AI 프롬프트"를 결합한 서비스는 현재 시장에 존재하지 않는다.

### 최근 가능해진 것

- AI 에이전트의 코드 생성 품질이 높아져, 좋은 프롬프트만 있으면 실무 수준의 UI 코드를 바로 얻을 수 있게 되었다.
- 프론트엔드 컴포넌트 패턴이 React/Vue/Svelte 등 프레임워크 간에 충분히 표준화되어, 프레임워크에 구애받지 않는 가이드가 가능해졌다.

## 4. Objective

### 목표

프론트엔드 컴포넌트의 구조와 용어를 **시각적으로** 이해하고, AI 코딩 툴에 바로 활용할 수 있는 **프롬프트를 즉시 복사**할 수 있게 한다.

### 왜 중요한가?

- **사용자 가치**: 컴포넌트 이름을 몰라도 와이어프레임을 보고 구조를 파악할 수 있다. AI 프롬프트를 직접 작성할 필요 없이 복사만 하면 된다.
- **비즈니스 가치**: AI 코딩 툴 생태계에서 "프롬프트 허브" 포지션을 선점할 수 있다. 검색 유입을 통한 오가닉 트래픽 확보가 가능하다.

### Key Results (OKR)

| # | Key Result | 측정 방식 |
|---|-----------|----------|
| KR1 | 출시 후 4주 내 주간 활성 사용자(WAU) 500명 달성 | Google Analytics |
| KR2 | 프롬프트 복사 버튼 클릭률 40% 이상 (페이지 방문 대비) | 이벤트 트래킹 |
| KR3 | 33개 컴포넌트 가이드 콘텐츠 100% 완성 | 콘텐츠 체크리스트 |
| KR4 | 평균 세션 체류 시간 3분 이상 | Google Analytics |

## 5. Market Segment(s)

### 핵심 사용자

| 세그먼트 | 해결하려는 문제 (Job-to-be-Done) | 규모 |
|---------|-------------------------------|------|
| **AI 코딩 툴 사용자** | "AI에게 UI를 요청하고 싶은데, 컴포넌트 이름과 구조를 몰라서 프롬프트를 못 쓰겠다" | 급성장 중 (Claude Code, Codex, GeminiCLI 사용자) |
| **프론트엔드 입문자** | "컴포넌트가 너무 많아서 뭐가 뭔지 모르겠다. 한눈에 정리된 자료가 필요하다" | 부트캠프 수강생, 독학 개발자 |
| **비개발 직군** | "개발팀과 소통할 때 프론트 용어를 몰라 답답하다" | 기획자, 디자이너, PM |

### 제약 조건

- 한국어 사용자를 1차 타겟으로 한다.
- 로그인 없이 즉시 사용 가능해야 한다 (진입 장벽 최소화).
- 데스크톱 환경을 우선 지원하고, 태블릿은 대응 수준으로 한다.

## 6. Value Proposition(s)

### 고객이 해결하려는 일 (Jobs)

1. UI 컴포넌트의 **정확한 명칭**을 알고 싶다.
2. 컴포넌트가 **페이지 내 어디에 위치하는지** 시각적으로 확인하고 싶다.
3. AI 에이전트에게 **바로 쓸 수 있는 프롬프트**를 얻고 싶다.
4. 팀과 소통할 때 **공통 용어**를 쓰고 싶다.

### 고객이 얻는 것 (Gains)

- 와이어프레임을 보며 컴포넌트 구조를 **직관적으로** 이해한다.
- 프롬프트 복사 한 번으로 AI 에이전트에게 **즉시 요청**할 수 있다.
- 핵심 용어 4개를 카드 형태로 **빠르게 학습**한다.

### 고객이 피하는 고통 (Pains)

- 컴포넌트 이름을 몰라 구글링에 시간을 낭비하는 일이 줄어든다.
- AI 프롬프트를 처음부터 직접 작성하는 시행착오가 사라진다.
- 영어 문서만 있어 이해하기 어려웠던 진입 장벽이 낮아진다.

### 경쟁 대비 차별점

| 기존 대안 | 한계 | FrontGuidKit 차별점 |
|----------|------|-------------------|
| MUI / Chakra 공식 문서 | 개발자 대상, 코드 중심, 영어 | 비개발자도 이해할 수 있는 시각 가이드 + 한국어 |
| UI 디자인 패턴 블로그 | 파편적, AI 프롬프트 없음 | 33개 컴포넌트 체계적 정리 + 프롬프트 제공 |
| AI 프롬프트 마켓플레이스 | 범용 프롬프트, UI 특화 아님 | 프론트엔드 컴포넌트에 특화된 프롬프트 |

## 7. Solution

### 7.1 UX / 화면 구조

```
┌─────────────────────────────────────────────────────┐
│  🔷 FrontGuidKit — 프론트엔드 개발 가이드 & AI 프롬프트  │  ← 상단바
├──────────┬──────────────────────────────────────────┤
│ 사이드바   │  메인 콘텐츠 영역                          │
│          │                                          │
│ 🏗️ 레이아웃 │  ┌──────────────────────────────────┐  │
│  · Header │  │ 컴포넌트 제목 + 한 줄 설명           │  │
│  · Hero   │  └──────────────────────────────────┘  │
│  · Navbar │                                          │
│  · Body   │  ┌──────────────────────────────────┐  │
│  · Footer │  │ 와이어프레임 미리보기 패널            │  │
│          │  │ (전체 페이지 내 위치 하이라이트)       │  │
│ 🔔 피드백  │  └──────────────────────────────────┘  │
│  · Modal  │                                          │
│  · Toast  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐     │
│  · Alert  │  │용어1 │ │용어2 │ │용어3 │ │용어4 │     │
│  · Badge  │  └─────┘ └─────┘ └─────┘ └─────┘     │
│  · ...    │                                          │
│          │  ┌──────────────────────────────────┐  │
│ 🧭 탐색   │  │ AI 프롬프트 패널                     │  │
│  · ...    │  │                          [복사] 📋  │  │
│          │  └──────────────────────────────────┘  │
│ 🃏 콘텐츠  │                                          │
│ 🎛️ 입력   │                                          │
│ 📊 시각화  │                                          │
└──────────┴──────────────────────────────────────────┘
```

**사용자 흐름:**
1. 사이드바에서 카테고리를 펼치고 컴포넌트를 선택한다.
2. 메인 영역에 해당 컴포넌트의 와이어프레임이 표시된다.
3. 와이어프레임 아래에서 핵심 용어 4개를 카드로 확인한다.
4. AI 프롬프트 패널에서 [복사] 버튼을 클릭해 프롬프트를 가져간다.

### 7.2 Key Features

#### F1. 사이드바 네비게이션
- 6개 카테고리로 33개 컴포넌트를 그룹핑하여 표시한다.
- 카테고리 클릭 시 하위 컴포넌트 목록이 펼쳐진다.
- 선택된 컴포넌트는 시각적으로 활성 상태를 표시한다.
- 컴포넌트 클릭 시 메인 영역이 즉시 전환된다 (CSR).

#### F2. 와이어프레임 미리보기
- 전체 페이지 레이아웃 안에서 해당 컴포넌트의 위치를 색상 하이라이트로 표시한다.
- 컴포넌트 내부의 세부 구성 요소(로고, 네비, CTA 등)를 텍스트 레이블로 표시한다.
- 실제 코드 없이 시각적 구조만 전달하는 다이어그램 형태이다.

#### F3. 핵심 용어 카드 (4개)
- 각 컴포넌트에 관련된 핵심 용어 4개를 카드 UI로 제공한다.
- 영어 명칭 + 한국어 설명 형태이다.
- 예: `Sticky Header` — 스크롤 시 상단에 고정되는 헤더

#### F4. AI 프롬프트 복사
- 컴포넌트별 AI 에이전트 구현 요청용 프롬프트 템플릿을 제공한다.
- 프롬프트에 구조, 동작, 스타일 등 핵심 항목이 포함된다.
- [복사] 버튼 클릭 시 클립보드에 전체 프롬프트가 복사된다.
- 복사 완료 시 버튼 상태가 "복사됨 ✓"으로 변경된다.

### 7.3 Technology

| 항목 | 선택 | 근거 |
|------|------|------|
| 프레임워크 | React (Vite) 또는 Next.js | CSR 중심, 빠른 페이지 전환 |
| 스타일링 | Tailwind CSS | 다크 테마 구현 용이, 빠른 개발 |
| 배포 | Vercel 또는 Netlify | 정적 사이트 무료 호스팅 |
| 콘텐츠 관리 | JSON/MDX 파일 기반 | DB 불필요, 정적 콘텐츠 |
| 클립보드 | Clipboard API | 프롬프트 복사 기능 |

### 7.4 Assumptions

| # | 가정 | 검증 방법 |
|---|------|----------|
| A1 | AI 코딩 툴 사용자는 컴포넌트 용어를 몰라서 프롬프트 작성에 어려움을 겪는다 | 사용자 인터뷰, 커뮤니티 설문 |
| A2 | 와이어프레임이 코드 예시보다 입문자에게 더 직관적이다 | A/B 테스트 (와이어프레임 vs 코드 스니펫) |
| A3 | 프롬프트 템플릿이 사용자마다 다른 AI 도구에서도 잘 동작한다 | Claude Code, Codex, GeminiCLI에서 프롬프트 테스트 |
| A4 | 33개 컴포넌트가 입문자가 마주치는 UI 패턴의 대부분을 커버한다 | 주요 웹사이트 분석, 사용자 피드백 |

## 8. Release

### Phase 1 — MVP (약 4-6주)

**범위:**
- 사이드바 네비게이션 (6개 카테고리, 33개 컴포넌트 목록)
- 레이아웃 카테고리 5개 컴포넌트 상세 가이드 완성 (Header, Hero, Navbar, Body, Footer)
- 와이어프레임 미리보기, 핵심 용어 4개, AI 프롬프트 복사 기능
- 다크 테마 UI
- 데스크톱 반응형

**목표:** 핵심 기능 검증 및 초기 사용자 피드백 수집

### Phase 2 — 콘텐츠 확장 (MVP 이후 4-6주)

**범위:**
- 나머지 28개 컴포넌트 가이드 콘텐츠 완성
- 피드백/알림, 탐색/이동, 콘텐츠 표시, 입력/인터랙션, 데이터 시각화 카테고리
- 태블릿 반응형 대응
- 사용자 피드백 기반 UX 개선

### Phase 3 — 고도화 (추후 검토)

**범위:**
- 추가 컴포넌트: Pricing Table, Testimonial, Feature Section, CTA Section, Cookie Banner
- 검색 기능 (컴포넌트명/용어 검색)
- 프롬프트 커스터마이징 (프레임워크 선택, 스타일 옵션)
- 다국어 지원 (영어)
- 커뮤니티 기능 (사용자 프롬프트 공유)
